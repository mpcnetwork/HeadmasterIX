MySQL databases

Werken met phpMyAdmin

Introductie

De databaseomgeving waarin we werken is phpMyAdmin. Hier voeren we MySQL-commando’s uit op verschillende databases. Manipulatie via SQL kan onomkeerbare gevolgen hebben, zoals het verwijderen van tabellen of gebruikers. Data is waardevol, en kennis van SQL is essentieel — zeker voor wie ethisch hacker wil worden.

Openen van phpMyAdmin

  1. Dubbelklik op het WampServer-icoon op je bureaublad.
  2. Start Apache en MySQL. Je pc fungeert nu als lokale webserver.
  3. Controleer het icoon in de taakbalk:
    • Groen: alles correct gestart
    • Oranje: één service niet gestart
    • Rood: niets gestart
  4. Klik op het groene icoon en kies phpMyAdmin.

Inloggen

  • Gebruikersnaam: root
  • Wachtwoord: leeg

Klik op Go om in te loggen. In een online omgeving moet je uiteraard een sterk wachtwoord instellen.

Overzicht van de omgeving

  • Server: MySQL via TCP/IP
  • Versie: MySQL 5.7.26
  • User: root@localhost
  • Charset: UTF-8 Unicode
  • Webserver: Apache 2.4.39 met PHP 7.3.5
  • phpMyAdmin versie: 4.8.5

Nieuwe database aanmaken

  1. Klik links op New.
  2. Geef een naam op (bijv. wagens).
  3. Kies utf8_general_ci als characterset.
  4. Klik op Create.

De standaard engine is MyISAM, maar deze ondersteunt geen relaties. Wijzig dit naar InnoDB via het bestand my.ini.

Engine wijzigen naar InnoDB

Open my.ini in C:\wamp64\bin\mysql\mysql5.7.26 en wijzig: 

; The default storage engine
default-storage-engine=InnoDB

Herstart WampServer om de wijziging toe te passen.

Database verwijderen

  1. Klik bovenaan op Server: MySQL:3306.
  2. Ga naar het tabblad Databases.
  3. Selecteer de database en klik op Drop.

Sample databases

  • sakila (deze gebruiken wij)
  • menagerie

Download de sakila database en pak uit op C:\sakila-db.

Installatie van Sakila database

  1. Open de MySQL-console via WampServer.
  2. Log in met root en druk op Enter (wachtwoord is blanco).
  3. Voer de volgende commando’s uit: 
    mysql> SOURCE C:/sakila-db/sakila-schema.sql;
mysql> SOURCE C:/sakila-db/sakila-data.sql;

Resultaat in phpMyAdmin

Na installatie zie je de sakila database in phpMyAdmin, met tabellen zoals:

  • actor
  • address
  • customer
  • film
  • rental
  • ...en vele andere

Alle tabellen zijn correct ingesteld met InnoDB en utf8mb4_general_ci als collation.

Tabellen en structuur in MySQL

Wat is een tabel?

Een tabel is een gestructureerde verzameling van gegevens binnen een database. Elke tabel bestaat uit rijen (records) en kolommen (velden). Elke kolom heeft een naam, een datatype en optionele instellingen zoals NULL, PRIMARY KEY, AUTO_INCREMENT, enz.

Een nieuwe tabel aanmaken

  1. Open phpMyAdmin en selecteer een database.
  2. Geef een naam op voor je nieuwe tabel (bijv. klanten).
  3. Geef het aantal kolommen op (bijv. 4).
  4. Klik op Go om de structuur te definiëren.

Datatypes in MySQL

Elke kolom in een tabel heeft een datatype dat bepaalt welk soort gegevens erin mogen staan:

  • INT: gehele getallen (bijv. leeftijd, ID)
  • VARCHAR(n): tekst met maximum n tekens (bijv. naam, e-mail)
  • TEXT: lange tekstvelden
  • DATE: datum (YYYY-MM-DD)
  • DATETIME: datum en tijd
  • BOOLEAN: waar/onwaar (true/false)
  • DECIMAL(x,y): decimale getallen met x cijfers waarvan y na de komma

Veldinstellingen

Bij het definiëren van kolommen kun je extra instellingen toevoegen:

  • NULL / NOT NULL: bepaalt of het veld leeg mag blijven
  • DEFAULT: standaardwaarde indien geen input
  • AUTO_INCREMENT: verhoogt automatisch bij elke nieuwe rij (vaak bij ID's)
  • UNIQUE: geen dubbele waarden toegestaan
  • COMMENT: optionele beschrijving van het veld

Primary Key

Een Primary Key is een unieke identificatie voor elke rij in een tabel. Meestal is dit een kolom zoals id met AUTO_INCREMENT. Een tabel mag slechts één primary key hebben.

Foreign Key

Een Foreign Key legt een relatie tussen twee tabellen. Het verwijst naar een Primary Key in een andere tabel. Dit zorgt voor referentiële integriteit.

Voorbeeld:

  • Tabel klanten heeft id als primary key.
  • Tabel bestellingen heeft klant_id als foreign key die verwijst naar klanten.id.

Relaties leggen in phpMyAdmin

  1. Ga naar het tabblad Structure van een tabel.
  2. Klik op Relation view onderaan.
  3. Selecteer de kolom die je wil koppelen als foreign key.
  4. Kies de gerelateerde tabel en kolom (meestal id).
  5. Activeer ON DELETE en ON UPDATE opties indien gewenst (bijv. CASCADE).

Best practices

  • Gebruik INT voor ID's en VARCHAR voor tekstvelden.
  • Stel NOT NULL in waar mogelijk om datakwaliteit te garanderen.
  • Gebruik InnoDB als engine voor relationele tabellen.
  • Geef duidelijke namen aan kolommen en tabellen (bijv. klant_id i.p.v. id2).

Veelgebruikte datatypes in MySQL

Bij het aanmaken van tabellen is het belangrijk om het juiste datatype te kiezen voor elke kolom. Hieronder vind je een overzicht van de meest gebruikte datatypes, inclusief hun bereik en typische toepassingen.

DECIMAL(x,y) Afhankelijk van x en y Afhankelijk van x en y Exacte decimale waarden (bv. prijzen)
FLOAT ±3.4E-38 tot ±3.4E+38 ±1.18E-38 tot ±3.4E+38 Benaderde decimale waarden
DOUBLE ±2.2E-308 tot ±1.8E+308 ±2.2E-308 tot ±1.8E+308 Wetenschappelijke berekeningen

BOOLEAN / BOOL 0 (false) of 1 (true) — Statusvelden, vinkjes

Tips bij datatypekeuze

  • Kies het kleinste datatype dat je nodig hebt voor efficiëntie.
  • Gebruik UNSIGNED als negatieve waarden niet nodig zijn.
  • Gebruik DECIMAL voor geldbedragen, niet FLOAT.
  • Gebruik VARCHAR in plaats van TEXT als je wilt sorteren of indexeren.
